## 3. 後端安裝（Supabase）

### 步驟 3.1：建立 Supabase 專案

1. 前往 https://supabase.com，登入或建立帳號
2. 點擊「**New Project**」
3. 填寫以下資訊：
   - **Project name**：`french-learning-app`
   - **Database password**：設定一個強密碼，**請妥善保存**
   - **Region**：選擇離你最近的區域（台灣用戶建議選 `ap-northeast-1` 東京 或 `ap-southeast-1` 新加坡）
4. 點擊「**Create new project**」，等待約 2 分鐘建立完成

### 步驟 3.2：取得 API 金鑰

專案建立後，進入「**Project Settings**」→「**API**」，記錄以下資訊：

```
Project URL    : https://xxxxxxxxxxxx.supabase.co
Anon Key       : eyJh...（公開金鑰，用於 Flutter App）
Service Role   : eyJh...（私密金鑰，只用於後端，勿洩露）
```

### 步驟 3.3：安裝 Supabase CLI

```bash
# macOS（使用 Homebrew）
brew install supabase/tap/supabase

# Windows（使用 Scoop）
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase

# Linux
curl -s https://raw.githubusercontent.com/supabase/supabase/master/scripts/install.sh | bash

# 驗證安裝
supabase --version
```

### 步驟 3.4：登入並連結專案

```bash
# 登入 Supabase
supabase login

# 進入後端目錄
cd french_learning_app/backend/supabase

# 初始化（若尚未初始化）
supabase init

# 連結到你的雲端專案
supabase link --project-ref YOUR_PROJECT_REF
# YOUR_PROJECT_REF 可在專案 URL 中找到：https://[THIS_PART].supabase.co
```

## 4. 資料庫初始化

請**依序**執行以下 SQL 腳本。可使用 Supabase Dashboard 的 SQL Editor 執行，或使用 CLI。

### 方法一：Supabase Dashboard（推薦給初次使用者）

1. 進入你的 Supabase 專案
2. 左側選單點擊「**SQL Editor**」
3. 點擊「**New query**」
4. 依序複製貼上以下檔案的內容並執行：

```
執行順序：
  1. database/01_schema_core.sql       ← 必須第一個執行
  2. database/02_schema_articles.sql
  3. database/03_schema_grammar.sql
  4. database/04_schema_notifications.sql
  5. database/05_seed_data.sql         ← 成就與範例課程
  6. backend/supabase/migrations/06_cron_jobs.sql
  7. database/07_migration_zhtw.sql    ← 繁體中文支援
  8. data/vocabulary_zh_tw_seed.sql    ← 300 字精校種子資料
```

> ⚠️ **注意：** 每個檔案必須完整執行，確認沒有錯誤後再執行下一個。

### 方法二：CLI 執行

```bash
cd french_learning_app

# 設定資料庫連線字串（從 Supabase Dashboard → Settings → Database 取得）
export DB_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT].supabase.co:5432/postgres"

# 依序執行
psql "$DB_URL" -f database/01_schema_core.sql
psql "$DB_URL" -f database/02_schema_articles.sql
psql "$DB_URL" -f database/03_schema_grammar.sql
psql "$DB_URL" -f database/04_schema_notifications.sql
psql "$DB_URL" -f database/05_seed_data.sql
psql "$DB_URL" -f backend/supabase/migrations/06_cron_jobs.sql
psql "$DB_URL" -f database/07_migration_zhtw.sql
psql "$DB_URL" -f data/vocabulary_zh_tw_seed.sql
```

### 步驟 4.1：確認資料表建立成功

在 SQL Editor 執行以下查詢，確認所有資料表存在：

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

預期結果應包含以下資料表：

```
achievements
articles
article_vocabulary
daily_articles
grammar_exercises
grammar_lessons
mistake_book
notification_log
notification_preferences
profiles
push_tokens
review_events
review_sessions
user_achievements
user_article_reads
user_exercise_attempts
user_grammar_progress
user_vocabulary_progress
vocabulary
```

### 步驟 4.2：啟用 pg_cron（定時任務）

1. 進入 Supabase Dashboard
2. 左側選單「**Database**」→「**Extensions**」
3. 搜尋「**pg_cron**」
4. 點擊切換開關啟用

## 5. 詞彙爬蟲執行

爬蟲會從 Lexique383 和英文維基詞典抓取 20,000 個法文單字，並透過 Google Translate 加入繁體中文翻譯。

### 步驟 5.1：安裝 Python 環境

```bash
cd french_learning_app/scraper

# 建立虛擬環境（推薦）
python3 -m venv .venv

# 啟動虛擬環境
# macOS / Linux：
source .venv/bin/activate
# Windows：
.venv\Scripts\activate

# 安裝依賴套件
pip install -r requirements.txt
```

### 步驟 5.2：設定環境變數

```bash
# 在 scraper/ 目錄下建立 .env 檔案
cat > .env << EOF
DATABASE_URL=postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT].supabase.co:5432/postgres
EOF
```

### 步驟 5.3：執行爬蟲

```bash
# 一鍵執行（macOS / Linux）
./run.sh

# 或手動執行（所有平台）
python main.py --limit 20000 --output both

# 同時寫入資料庫
python main.py --limit 20000 --output both --db
```

#### 執行參數說明

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `--limit N` | 目標單字數量 | 20000 |
| `--output sql` | 只輸出 SQL 檔案 | both |
| `--output csv` | 只輸出 CSV 檔案 | both |
| `--output both` | 同時輸出 SQL 和 CSV | both |
| `--db` | 同時寫入資料庫 | 不啟用 |

#### 預估執行時間

| 單字數 | 預估時間 |
|--------|----------|
| 5,000  | 約 15 分鐘 |
| 10,000 | 約 30 分鐘 |
| 20,000 | 約 60–90 分鐘 |

> **說明：** 大部分時間花在 Wiktionary API 請求（網路限速）和 Google Translate 翻譯。爬蟲有本地快取機制，中斷後重新執行會繼續上次進度，不會重複請求。

### 步驟 5.4：匯入 SQL 到資料庫

若執行時沒加 `--db`，爬蟲完成後請手動匯入：

```bash
# 方法一：psql
psql "$DB_URL" -f scraper/data/output/vocabulary_20000.sql

# 方法二：使用 CSV 批次匯入（速度更快）
psql "$DB_URL" << 'SQL'
\copy vocabulary (id, french_word, english_trans, translations, word_class,
                  gender, plural_form, pronunciation_ipa, cefr_level,
                  topic_tags, frequency_rank, example_sentences,
                  usage_notes, is_active)
FROM 'scraper/data/output/vocabulary_20000.csv'
WITH (FORMAT csv, HEADER true, QUOTE '"');
SQL
```

### 步驟 5.5：驗證資料數量

```sql
SELECT cefr_level, COUNT(*) as 單字數
FROM vocabulary
GROUP BY cefr_level
ORDER BY cefr_level;
```

預期結果（大約值）：

| cefr_level | 單字數 |
|------------|--------|
| A1 | ~500 |
| A2 | ~1,500 |
| B1 | ~3,000 |
| B2 | ~5,000 |
| C1 | ~5,000 |
| C2 | ~5,000 |

## 6. Flutter App 安裝

### 步驟 6.1：安裝 Flutter SDK

1. 前往 https://flutter.dev/docs/get-started/install
2. 選擇你的作業系統，依照官方說明安裝
3. 執行以下指令確認安裝成功：

```bash
flutter doctor
```

確保所有必要項目顯示 ✓ 綠色勾勾（至少需要 Flutter、Dart、目標平台的模擬器或連接的裝置）。

### 步驟 6.2：進入 Flutter 專案

```bash
cd french_learning_app/flutter_app
```

### 步驟 6.3：設定環境變數

在 `flutter_app/` 目錄建立 `.env` 或直接修改設定常數：

**方法一：編譯時帶入（推薦）**

建立 `flutter_app/.env`：
```
SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
SUPABASE_ANON_KEY=eyJh...你的 Anon Key...
```

**方法二：直接修改程式碼**

編輯 `lib/core/constants/app_constants.dart`：
```dart
static const supabaseUrl     = 'https://xxxxxxxxxxxx.supabase.co';
static const supabaseAnonKey = 'eyJh...你的 Anon Key...';
```

> ⚠️ **安全提醒：** Anon Key 是公開金鑰，可以放在 App 中。Service Role Key 絕對不可以放在前端。

### 步驟 6.4：下載字型檔案

下載繁體中文字型（Noto Sans TC）：

1. 前往 https://fonts.google.com/noto/specimen/Noto+Sans+TC
2. 點擊「Download family」下載壓縮檔
3. 解壓後將以下三個檔案複製到 `flutter_app/assets/fonts/`：
   - `NotoSansTC-Regular.ttf`
   - `NotoSansTC-Medium.ttf`
   - `NotoSansTC-Bold.ttf`

同樣下載 Spectral 字型：
1. 前往 https://fonts.google.com/specimen/Spectral
2. 下載並複製到 `flutter_app/assets/fonts/`：
   - `Spectral-Regular.ttf`
   - `Spectral-Italic.ttf`
   - `Spectral-Bold.ttf`

### 步驟 6.5：建立資源目錄

```bash
cd flutter_app
mkdir -p assets/fonts assets/images assets/animations assets/audio
```

### 步驟 6.6：安裝 Flutter 套件

```bash
cd flutter_app
flutter pub get
```

### 步驟 6.7：程式碼產生

本專案使用 Freezed（資料模型）和 Riverpod（狀態管理）的程式碼產生功能：

```bash

```

> 這個步驟大約需要 1–3 分鐘，執行完成後應產生 `.g.dart` 和 `.freezed.dart` 檔案。

### 步驟 6.8：執行 App


## 7. 推播通知設定

### 7.1 建立 Firebase 專案

1. 前往 https://console.firebase.google.com
2. 點擊「**新增專案**」，命名為 `french-learning-app`
3. 依照指示完成建立

### 7.2 Android 設定

1. Firebase Console → 新增 Android 應用程式
2. **Android 套件名稱**：`com.yourname.frenchlearning`（需與 `android/app/build.gradle` 一致）
3. 下載 `google-services.json`
4. 將檔案放置於 `flutter_app/android/app/google-services.json`

### 7.3 iOS 設定

1. Firebase Console → 新增 iOS 應用程式
2. **iOS 套件組合 ID**：`com.yourname.frenchlearning`
3. 下載 `GoogleService-Info.plist`
4. 將檔案放置於 `flutter_app/ios/Runner/GoogleService-Info.plist`
5. 以 Xcode 開啟 `ios/Runner.xcworkspace`，確認 Bundle ID 一致

### 7.4 設定 APNs（iOS 推播，需 Apple Developer 帳號）

1. Apple Developer Portal → Certificates, IDs & Profiles
2. 建立 APNs Key（或 Certificate）
3. Firebase Console → 你的 iOS App → Cloud Messaging → 上傳 APNs 金鑰

### 7.5 設定 Supabase 通知排程

在 Supabase SQL Editor 設定 Edge Function 基礎 URL：

```sql
ALTER DATABASE postgres
SET "app.edge_function_base_url" = 'https://xxxxxxxxxxxx.supabase.co/functions/v1';

ALTER DATABASE postgres
SET "app.cron_secret" = '你自訂的隨機密鑰';
```

### 7.6 部署通知 Edge Function

```bash
cd backend/supabase

# 設定密鑰
supabase secrets set CRON_SECRET=你自訂的隨機密鑰

# 部署通知函式
supabase functions deploy notifications
```

## 8. 本地開發啟動

若想在本機執行完整的後端（不使用雲端 Supabase），請按照以下步驟：

### 步驟 8.1：確認 Docker 已啟動

```bash
docker info
```

### 步驟 8.2：啟動本地 Supabase

```bash
cd french_learning_app/backend/supabase

# 啟動（第一次會下載 Docker 映像，需要幾分鐘）
supabase start
```

啟動成功後會顯示：

```
Started supabase local development setup.

         API URL: http://localhost:54321
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324      ← 本地測試電子郵件
        anon key: eyJh...（本地測試用 Anon Key）
service_role key: eyJh...（本地測試用 Service Role Key）
```

### 步驟 8.3：初始化本地資料庫

```bash
export DB_URL="postgresql://postgres:postgres@localhost:54322/postgres"

psql "$DB_URL" -f ../../database/01_schema_core.sql
psql "$DB_URL" -f ../../database/02_schema_articles.sql
psql "$DB_URL" -f ../../database/03_schema_grammar.sql
psql "$DB_URL" -f ../../database/04_schema_notifications.sql
psql "$DB_URL" -f ../../database/05_seed_data.sql
psql "$DB_URL" -f ../../database/07_migration_zhtw.sql
psql "$DB_URL" -f ../../data/vocabulary_zh_tw_seed.sql
```

### 步驟 8.4：啟動本地 Edge Functions

```bash
# 在一個終端機視窗執行
supabase functions serve
```

### 步驟 8.5：設定 Flutter 使用本地後端

修改 `flutter_app/lib/core/constants/app_constants.dart`：

```dart
static const supabaseUrl     = 'http://localhost:54321';
static const supabaseAnonKey = '本地 Anon Key（從 supabase start 輸出取得）';
```

### 步驟 8.6：開啟 Supabase Studio

```bash
# 瀏覽器開啟本地管理介面
open http://localhost:54323
```

在 Studio 可以：
- 查看和編輯資料表內容
- 執行 SQL 查詢
- 查看 Edge Function 日誌
- 管理使用者帳號

### 停止本地服務

```bash
supabase stop
```

## 9. 部署到正式環境

### 步驟 9.1：部署所有 Edge Functions

```bash
cd french_learning_app/backend/supabase

# 一次部署所有函式
supabase functions deploy flashcards
supabase functions deploy articles
supabase functions deploy grammar
supabase functions deploy mistake-book
```

```bash
supabase secrets set \
  CRON_SECRET="你的隨機密鑰" \
  GOOGLE_CLIENT_ID="Google OAuth 客戶端 ID（如有啟用）" \
  GOOGLE_SECRET="Google OAuth 密鑰（如有啟用）"
```

```bash
cd flutter_app
```

輸出位置：`build/app/outputs/flutter-apk/app-release.apk`

```bash

  --dart-define=SUPABASE_URL=https://xxx.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=eyJh...
```

輸出位置：`build/app/outputs/bundle/release/app-release.aab`

#### iOS（需 macOS + Xcode + Apple Developer 帳號）


## 11. 常見錯誤排解

### 問題 1：`flutter pub get` 失敗

**錯誤訊息：**
```
Because french_learning_app depends on riverpod_annotation >=2.3.5...
```

**解決方法：**
```bash
# 清除快取後重試
flutter clean
flutter pub cache repair
flutter pub get
```

---

### 問題 2：`build_runner` 產生失敗

**錯誤訊息：**
```
[SEVERE] Failed to generate build script
```

**解決方法：**
```bash
dart run build_runner clean
dart run build_runner build --delete-conflicting-outputs
```

---

### 問題 3：SQL 執行時出現「extension already exists」

**錯誤訊息：**
```
ERROR: extension "uuid-ossp" already exists
```

**解決方法：** 這是正常情況，Supabase 預設已安裝部分擴充套件。可以將相關行改為：
```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```
（本專案腳本已使用 `IF NOT EXISTS`，不應出現此錯誤）

---

### 問題 4：Flutter App 無法連接 Supabase

**症狀：** 登入畫面出現「無法連接伺服器」或永久載入

**解決方法：**
1. 確認 `SUPABASE_URL` 和 `SUPABASE_ANON_KEY` 正確
2. 確認手機或模擬器有網路連線
3. 使用本地開發時，確認模擬器網路設定（Android 模擬器使用 `10.0.2.2` 代替 `localhost`）

```dart
// Android 模擬器本地開發用
static const supabaseUrl = 'http://10.0.2.2:54321';
```

---

### 問題 5：爬蟲執行中斷

**症狀：** 爬蟲到一半網路中斷或程式崩潰

**解決方法：** 直接重新執行，爬蟲有快取機制：
```bash
python main.py --limit 20000 --output both
```
已抓取的 Wiktionary 資料和翻譯會從快取讀取，只會繼續未完成的部分。

---

### 問題 6：Google Translate 返回錯誤

**症狀：** 翻譯全部為空白，或出現 `429 Too Many Requests`

**解決方法：**
1. 等待 5–10 分鐘後重試（Google Translate 有頻率限制）
2. 降低並行數，修改 `scraper/config.py`：
   ```python
   TRANSLATE_WORKERS   = 2   # 從 5 改為 2
   TRANSLATE_DELAY_SEC = 1.0 # 從 0.30 改為 1.0
   ```
3. 重新執行（快取中已翻譯的不會重複請求）

---

### 問題 7：Edge Function 部署失敗

**錯誤訊息：**
```
Error: Failed to deploy function
```

**解決方法：**
```bash
# 確認已登入
supabase login

# 確認已連結專案
supabase link --project-ref YOUR_PROJECT_REF

# 查看詳細錯誤
supabase functions deploy flashcards --debug
```

---

### 問題 8：iOS 推播通知不生效

**解決方法：**
1. 確認 APNs Key 已上傳到 Firebase Console
2. 確認 App 的 Bundle ID 和 Firebase 設定一致
3. 真實裝置才能測試推播（模擬器不支援）
4. 確認 Xcode 中已啟用 Push Notifications Capability

---

### 問題 9：`supabase start` 失敗

**錯誤訊息：**
```
Cannot connect to the Docker daemon
```

**解決方法：** 確認 Docker Desktop 已開啟並正在執行，然後重試：
```bash
# 確認 Docker 狀態
docker info

# 重新啟動 Docker Desktop 後再試
supabase start
```

---

### 問題 10：繁體中文字型顯示為方框（豆腐字）

**症狀：** App 內繁體中文顯示為 □□□

**解決方法：**
1. 確認字型檔案已放置在正確路徑
2. 確認 `pubspec.yaml` 中字型路徑正確
3. 重新建置：
   ```bash
   flutter clean
   flutter pub get
   flutter run
   ```
